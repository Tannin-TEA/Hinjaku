# Gemini-MEMO

## 画像表示・解像度制限
- **最大デコード解像度**: 4K (4096px) - `MAX_IMAGE_DIMENSION`
  - これを超える画像はデコード後にこのサイズまで縮小する。
- **最大テクスチャサイズ**: Full HD (1920px) - `MAX_TEX_DIM`
  - VRAM節約のため、表示用データはFull HD相当を上限とする。

## パフォーマンス・軽量化設定
- **レンダラー**: `eframe::Renderer::Glow` (OpenGL) を採用。
  - `wgpu` よりも初期起動時のメモリフットプリント（Working Set）を抑制する。
- **GPUリソース**: `multisampling`, `depth_buffer`, `stencil_buffer` はすべて `0` (無効)。
- **並列処理**: デコードワーカースレッドは `2` (`WORKER_THREADS`)。
- **キャッシュ**: 画像キャッシュ上限は `8` (`CACHE_MAX`)、ツリーノードキャッシュは `500`。
- **フォント**: メモリ節約のため、Windows標準の `msgothic.ttc` などを優先的に使用し、読み込みは最小限にする。

## 機能実装の指針
- **常に手前に表示**: `config.ini` の `AlwaysOnTop` で永続化。`egui::WindowLevel::AlwaysOnTop` で制御。
- **末尾から開く**: `config.ini` の `OpenFromEnd` で管理。マンガモード（見開き）時のインデックス計算を厳密に行う。
- **二重起動防止**: Windows Mutex を使用。

## IPC・サーバー関連実装の廃止
- 名前付きパイプ（Named Pipe）を利用した「サーバー/クライアント」的なパス受け渡しコードをすべて削除。
- 個人用デスクトップアプリとして不適切かつ不安定な要素を排除。
- 今後、既存プロセスへのパス通知が必要な場合は、Windowsメッセージ（WM_COPYDATA等）などのよりネイティブかつ簡潔な手法を検討すること。

## 開発環境・ビルド
- Windows環境での動作を優先。
- `windows-sys` クレートを使用し、直接 Win32 API を叩くことで軽量化を図る。
- リリースビルド時は `lto = true`, `strip = true` を適用。
