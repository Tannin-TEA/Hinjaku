# Hinjaku

Rust/egui で構築された、軽量で高速な画像アーカイブビューアです。
**吹けば飛ぶよな軽量ビューア**

## Disclaimer
Hinjaku は、Windows 環境に特化した、極めて軽量で高速な画像ビューアです。  
アーカイブ（ZIP/7z）内の画像を、一時ファイルを作らずに直接ストリーム読み込みすることで、低メモリ消費と高速な閲覧を実現しています。

本ソフトウェアは「現状のまま」提供されます。
明示・黙示を問わず、特定目的への適合性や正確性について一切保証しません。
利用により生じたいかなる損害についても、作者は責任を負いません。
## 主な特徴

## 主な機能
- **対応フォーマット**: ZIP, 7z, 通常のフォルダ, 単一画像ファイル。
- **対応画像形式**: JPG, PNG, WEBP, GIF, BMP, TIFF。
- **表示モード**:
  - 1ページ表示 / マンガモード（2ページ見開き）。
  - 右開き (RTL) / 左開き対応。
  - フィット(Fit) / ウィンドウサイズに合わせる(WindowFit) / 100%(Original)。
  - フルスクリーン / ボーダレス最大化。
  - 透明画像の背景選択 (アプリ既定 / 市松模様 / 黒 / 灰 / 白)。
- **ナビゲーション**:
  - サイドバーによるディレクトリツリー表示。
  - フォルダ間移動（前/次のフォルダへ）。
  - 自然順（数値考慮）ソートを含む多角的なソート機能。
- **外部連携**: 任意の外部アプリで表示中の画像を開く機能。
- **Windows 専用設計**: Windows API や `ShellExecuteW` を活用し、OS と親和性の高い動作をします。
- **アーカイブ直接閲覧**: ZIP および 7z 形式に対応。解凍の手間なく中身を閲覧可能です。
- **外部アプリ連携 (ActionKey)**: 表示中のファイルパスを Photoshop などの外部ツールへ即座に渡すことができます。
- **ゼロ・テンポラリ**: アーカイブ展開時にディスクへ一時ファイルを一切書き込みません。
- **INI ベース設定**: 設定はすべて `config.ini` で管理され、ポータブルに使用できます。

## コマンドラインオプション

### 1. ナビゲーション・ガード (誤操作防止)
ページめくり時の「意図しない連打」を防ぐため、以下のロック時間が設定されています。
- **フォルダ・アーカイブ移動ロック**: `0.5秒` (移動直後、または PgUp/PgDn 等で新しいフォルダに入った最初の画像が表示されてからロック)
- **マンガモード揃えロック**: `0.05秒` (マンガモードで左右のページが揃った瞬間の誤操作防止ロック)
- **マウスホイールしきい値**: `40.0` (感度調整用。一定量のスクロールが蓄積されるまでページ移動しません)
起動時に以下の引数を指定することで、動作をカスタマイズできます。

### 2. パフォーマンスとメモリ管理
- **画像キャッシュ**: 最大 `13枚`。
- **プリフェッチ**: 現在のページから前方 `5枚`、後方 `5枚` をバックグラウンドで事前読み込み。
- **自動リサイズ**: 解像度が `4096px` を超える画像は、GPU転送前に自動的にダウンスケールされます。
- **アニメーション制限**: GIF/WebP アニメーションは、ファイルサイズが `30MB` 以下のもののみ再生されます。
- **スタッター防止**: GPUへのテクスチャ転送は1フレームにつき最大 `2枚` に制限し、操作の引っかかりを防止します。
```bash
hinjaku.exe [パス] [-c 設定名] [--debug]
```

### 3. マンガモードのインテリジェンス
- 見開き画像（横長画像）を検出すると、自動的に1枚で表示します。
- フォルダの1枚目（表紙想定）は常に1枚で表示し、2枚目から見開きを開始します。
- 左右のページが両方ロードされるまで表示更新を待機し、ガタつきを防止します。
- **`パス`**: 起動時に開く画像ファイル、フォルダ、またはアーカイブファイルのパスを指定します。
- **`-c`, `--config`**: 使用する設定ファイル（INI）のプロファイル名を指定します。  
  例: `-c sub` と指定すると `sub.ini` を読み込みます。指定がない場合は `config.ini` が使用されます。
- **`--debug`**: コンソールにメモリ使用量やキャッシュ統計などのデバッグログを出力します。

## 使用ライブラリ

## 設定 (config.ini)
以下の項目が `config.ini` (TOML形式) で保存されます。
- `external_app`: 外部アプリのパス。
- `external_args`: 引数（`%P` がファイルパスに置換されます）。
- `linear_filter`: 画像補正のデフォルト設定。
- `allow_multiple_instances`: 複数起動の許可設定。
- `sort_mode` / `sort_order` / `sort_natural`: ソート設定の永続化。
- `manga_rtl`: 右開きのデフォルト設定。
本プロジェクトは、以下の素晴らしいオープンソースライブラリを使用して開発されています。

- **eframe / egui**: GUI フレームワーク
- **image**: 画像デコード処理
- **zip-rs**: ZIP アーカイブ操作
- **sevenz-rust**: 7z アーカイブ操作
- **rust-ini**: INI 設定ファイルの読み書き
- **windows-sys**: Windows API 連携
- **rfd**: ネイティブファイルダイアログ

## 開発憲法 (Immutable Rules)

1. **Windows Native Only**: Windows 環境に特化し、UNC パスや OS 固有機能を優先する。
2. **INI-Based Config**: 設定は INI 形式を厳守する。JSON/TOML は採用しない。
3. **Zero-Temp Strategy**: ディスクへの一時ファイル作成を禁止し、メモリ内で完結させる。
4. **No-Code Friendly**: 初心者でも理解しやすい具体的な命名と日本語コメントを維持する。
