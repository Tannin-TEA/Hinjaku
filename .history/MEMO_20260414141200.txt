■ 基本的な操作方法

1. GitHubからプル（最新の状態を取得）
   自分や他の人がアップロードした最新のコードを、自分のPCに取り込む操作です。
   コマンド: git pull origin main
   ※ origin: GitHub（リモート）の場所
   ※ main: メインのブランチ名

2. Cargo（Rustのビルドツール）の基本操作
   ・ビルド（開発用）: cargo build
     開発中に使用。コンパイルは速いが、実行速度は遅い。
   ・ビルド（本番用）: cargo build --release
     最適化された高速なバイナリを作成。target\release\hinjaku.exe が生成される。
   ・実行: cargo run
     コンパイルと実行をセットで行う。
   ・掃除: cargo clean
     ビルドで作られた一時ファイルを消してディスク容量を空ける。

■ 現在の主な追加・変更点まとめ

1. 世代管理（Generation）の導入
   アーカイブを切り替えるたびに「世代番号」を更新。
   古いアーカイブの読み込み結果が遅れて届いても、現在の表示を汚したり、
   「読み込み中」のままフリーズしたりするのを防ぎます。

2. キャッシュキーのユニーク化
   識別キーを「インデックス:アーカイブパス:ファイル名」に変更。
   別のアーカイブにある同名ファイル（001.jpgなど）を混同する問題を解消しました。

3. Pending状態の解消（スキップ通知）
   読み込みスレッドが「遠すぎる」と判断してスキップした際も、メインスレッドへ通知。
   「リクエスト中（Pending）」フラグが残り続けてロードが止まる現象を修正しました。

4. エラーハンドリングの可視化
   デコード失敗などの際、ただ止まるのではなく「⚠ エラー内容」を赤字で表示。
   何が原因で表示できないのかが分かるようになりました。

5. アニメーション対応の強化
   GIF/WebPアニメーションのデコード処理を改善。
   30MBのサイズ制限や、0msなどの極端に短いフレーム遅延の補正ロジックを追加し、
   アプリのクラッシュやフリーズを防止しています。

6. ソート（並べ替え）時の不整合修正
   Sキーでのソート変更時、即座にキャッシュをクリア。
   古い並び順のインデックスで読み込みがループするバグを修正しました。

7. 外部アプリ連携の改善
   Eキーで開く際のパス構築を改善。スペースを含むパスや、アーカイブ内ファイルの指定精度を高めました。
